FAIR and FAIRification

FAIR(Findability, Accessibility, Interoperability, and Reusability) principles serve to guide data producers and data publishers to improve their data management practices and improve the reuse of research data[1] FAIRification is the process of improving the FAIR level of datasets, projects or related infrastructure.

Challenges in FAIRification

1. While many people are aware of FAIR, it is hard to find the ideal state and FAIRification goals for their projects.
2. Implementing FAIR requires a broad spectrum of knowledge and skills.

There is no single platform providing structured guidance and systematic training in one place.

FAIR Wizard

The FAIR Wizard allows people without specific prior knowledge to explore solutions to help them adopt the FAIR principles, as well as practical steps they can take to do so, supported by contextually-relevant documentation and tools. It also connects different FAIR communities and serves as a unified entrance for finding resources related to FAIR.

Aims

1. To identify FAIRification goals through a list of questions or FAIR assessment.
2. To recommend solutions for implementing FAIR based on user requirements

Example FAIRification goals

Summary Description
Identifying chemical compounds for data exchange As a data owner, I want to use appropriate identifiers for the assay data of chemical compounds, so that I can connect all data about the same compound and exchange the data with others.
Applying community metadata standards As a biologist, I want to use community metadata standards for data planning so that I can avoid missing information during data collection and adopt the communication exchange standards before the data is born.
Using vocabularies to annotate data As a curator, I want to annotate the metadata with standard vocabularies, so that the data can be consistent and more interoperable.
